# MAPS Final Translation Extraction Failure

## Symptom

MAPS (paper strategy) COMET score was **0.7194**, lower than zero-shot's **0.7509**, even though the 3-stage human-like translation approach should theoretically outperform direct prompting. The gap was −0.0315 (−4.19%).

## Root Cause

The MAPS prompt uses bold markdown headers for its steps (`**Step 1 —`, `**Step 2 —`, `**Step 3 —`). Qwen mirrors this formatting in its response and writes the final marker as:

```
**FINAL TRANSLATION:**
Bu çevirinin son halidir.
```

The original `extract_final_translation` function had three bugs that all stem from this mismatch:

### Bug 1 — Bold markers not stripped (caused `****` outputs)

```python
# old code
if "FINAL TRANSLATION:" in line:
    return line.replace("FINAL TRANSLATION:", "").strip()
```

When the line was `"**FINAL TRANSLATION:**"`, `replace` left `"** **"` → strip → `"****"`. This garbage string was then scored against Turkish references, collapsing COMET to near zero for those sentences.

### Bug 2 — Translation on the next line, silently missed

The extractor scanned lines in **reverse**. When Qwen placed the translation on the line *after* the marker:
```
**FINAL TRANSLATION:**
Bu çevirinin son halidir.
```
Reversed scan hit `"Bu çevirinin son halidir."` first (no marker match), then hit `"**FINAL TRANSLATION:**"` (marker match), returned `"****"` — the actual translation was never seen.

### Bug 3 — Case-sensitive match

`"FINAL TRANSLATION:" in line` failed silently when Qwen wrote `"Final Translation:"` (title case). The fallback then returned the **last non-empty line of the entire response**, which could be a line from the Step 1 analysis or Step 3 refinement commentary — not the translation itself.

## Scale of Impact

| Category | Count | % |
|---|---|---|
| `****` (only stars) | 27 | 13.5% |
| Empty string | 3 | 1.5% |
| `** text` (stars + text) | 10 | 5.0% |
| Clean extraction | 160 | 80.0% |

**20% of translations were garbage.** The 160 clean MAPS translations score ~0.82 COMET (genuinely better than zero-shot). The 40 failures dragged the system average to 0.7194.

## Fix

**File:** `modules/prompts.py` — `extract_final_translation`

```python
def extract_final_translation(output: str) -> str:
    import re
    lines = output.strip().splitlines()

    for i, line in enumerate(lines):
        if re.search(r"FINAL TRANSLATION\s*:", line, re.IGNORECASE):
            after = re.split(r"FINAL TRANSLATION\s*:", line, maxsplit=1, flags=re.IGNORECASE)[1]
            after = re.sub(r"^\s*\*+\s*", "", after)
            after = re.sub(r"\s*\*+\s*$", "", after).strip()
            if after:
                return after
            # Translation is on the next line
            if i + 1 < len(lines):
                next_line = re.sub(r"^\s*\*+\s*", "", lines[i + 1]).strip()
                if next_line:
                    return next_line

    # Fallback: last non-empty, non-markdown line
    for line in reversed(lines):
        stripped = line.strip()
        if stripped and not re.fullmatch(r"[\*\s\-_#:]+", stripped):
            return stripped

    return output.strip()
```

Changes:
- `re.IGNORECASE` — catches `Final Translation:`, `FINAL TRANSLATION:`, etc.
- `re.split` on the marker — then strips `**` from both edges of the extracted text
- Look-ahead — if extracted text is empty, takes the next line
- Fallback skips pure-markdown lines (`****`, `---`, `**`) instead of returning them

**File:** `modules/translation.py` — `translate_paper_strategy`

`max_new_tokens` raised from **512 → 768** to prevent truncation before the `FINAL TRANSLATION:` line on complex, long sentences.
